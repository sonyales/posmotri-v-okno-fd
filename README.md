# Посмотри в окно

Интерактивный веб-интерфейс для поиска и просмотра видео по выбранному городу и времени суток. Проект реализован на HTML, CSS и JavaScript без использования сторонних фреймворков.

## Описание

Проект представляет собой страницу с видеоплеером, формой поиска и списком карточек с результатами. Пользователь может искать видео по названию города, выбирать время суток, переключаться между найденными вариантами и подгружать дополнительные результаты.

Основная логика построена вокруг работы с внешним API: приложение отправляет запросы с параметрами фильтрации, получает данные, генерирует карточки видео и обновляет основной видеоплеер без перезагрузки страницы.

## Функциональность

- загрузка видео с внешнего API;
- поиск видео по городу;
- фильтрация результатов по времени суток;
- отображение основного видео;
- генерация списка карточек на основе полученных данных;
- переключение активного видео по клику на карточку;
- визуальное выделение выбранной карточки;
- подгрузка дополнительных результатов через кнопку «Показать еще»;
- отображение прелоадера во время загрузки данных;
- обработка состояния, когда подходящие видео не найдены;
- обработка ошибок при получении данных;
- кастомный скроллбар для списка карточек;
- стилизованные состояния кнопок, полей формы и интерактивных элементов.

## Особенности реализации

### HTML

Разметка построена на семантической структуре с формой поиска, областью основного видео и списком результатов. Для повторяющихся элементов используются HTML-шаблоны, на основе которых JavaScript динамически создаёт карточки, прелоадеры, сообщения об ошибке и кнопку подгрузки дополнительных данных.

Форма содержит поле для ввода города и чекбоксы для выбора времени суток. После отправки формы страница не перезагружается: данные обрабатываются через JavaScript.

### CSS

Стили проекта разделены по назначению:

- `style.css` — основная композиция страницы, сетка, форма, карточки, кнопки и состояния элементов;
- `preloader.css` — стилизация прелоадера;
- `error.css` — стилизация блока ошибки.

Интерфейс построен на двухколоночной сетке: слева расположен видеоплеер и форма поиска, справа — заголовок и список найденных видео. Для компоновки используются CSS Grid и Flexbox.

Отдельно проработаны состояния интерактивных элементов: наведение, активное состояние, фокус с клавиатуры и выбранная карточка.

### JavaScript

JavaScript отвечает за основную механику приложения:

- отправку запросов к API;
- сериализацию данных формы;
- формирование URL с параметрами фильтрации;
- получение и обработку данных;
- динамическое создание карточек видео;
- установку видео и постера в основной плеер;
- переключение текущего видео;
- отображение и удаление прелоадеров;
- обработку ошибок;
- реализацию кнопки «Показать еще»;
- работу с пагинацией.

Основная логика находится в файле:

```txt
scripts/script.js
```

## Работа с API

Приложение получает данные с внешнего API. На основе ответа сервера формируется список карточек, а первое видео из полученных результатов автоматически устанавливается в основной видеоплеер.

При отправке формы параметры поиска добавляются в запрос:

- город передаётся как текстовый фильтр;
- выбранное время суток передаётся как один или несколько параметров;
- пагинация используется для подгрузки следующих страниц результатов.

Если API не возвращает подходящие данные, интерфейс показывает сообщение об отсутствии видео.

## Как запустить проект локально

Склонируйте репозиторий:

```bash
git clone https://github.com/sonyales/posmotri-v-okno-fd.git
```

Перейдите в папку проекта:

```bash
cd posmotri-v-okno-fd
```

Откройте файл `index.html` в браузере.

Установка зависимостей и дополнительная сборка не требуются.
